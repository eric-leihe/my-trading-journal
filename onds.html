4. Core vs Trading 规则卡片

Rules

4.1 Core 仓位规则（长期）

  • 只因：行业 / 公司长期逻辑明确 → 建 / 加仓
  • 可承受：–30% / –40% 的长周期波动
  • 卖出只看三件事：
  • ① 行业需求反转（无人机防卫预算长期下滑）
  • ② 公司执行失败（收入低于指引、订单/收购明显恶化）
  • ③ 资本结构失控（反复严重摊薄、融资质量恶化）
波动≠错误，逻辑破坏才动

4.2 Trading 仓位规则（右侧）

  • 只因：技术 + 叙事 + 资金 三动能共振 → 入场
  • 绝不长期死扛，严格价格止损
  • 止损：跌破突破位 / 回撤低点 → 全部平仓
  • 止盈：大阳、上影、RSI 过热、偏离均线 → 分批兑现
  • Trading 仓位出局 ≠ 看空公司，只代表“动能交易这一轮结束”
错了就走，走了再看 拒绝：不设止损的 Trading 仓

5. 每次操作 ONDS 前的 30 秒 Checklist

Discipline

每次你想对 ONDS「加仓 / 减仓 / 止损 / 止盈」时，先在脑子里或纸上过一遍下面这段。

【ONDS 30 秒 Checklist】 1. 这笔是 Core 还是 Trading？ - Core：写上“长期无人机防卫逻辑”，不能再改成短线想法。 - Trading：写上“右侧动能波段”，必须有止损。 2. 如果是 Core： - 最近 1–2 周有无行业/公司实质性负面？ - 如果没有 → 不动仓位，只记录情绪与波动。 3. 如果是 Trading： - 当前是 突破买点 / 回撤买点 / 止盈点 / 止损点？ - 止损价格是多少？亏损额度是否可接受？ - 是否已经属于“第二天、第三天的大阳 FOMO 加仓”？若是 → 减半仓位执行，或直接不做。 4. ONDS 总占比是否仍在 10% 上限以内？ - 若超过 → 只减不加。
